# Load Generator

This directory contains the workload generation components used to drive traffic through both benchmark paths.

The load generator exists to provide consistent request patterns across the comparison environment.

Responsibilities may include:
- steady-state request generation
- burst traffic generation
- valid request simulation
- invalid request simulation
- randomized workload behavior
- concurrency testing
- repeatable benchmark execution

The purpose of the shared load generator is to ensure both paths experience equivalent traffic conditions during measurement and comparison.
