# Schemas

This directory contains shared request, response, event, and metric schema definitions used throughout the benchmark environment.

Schemas exist to normalize communication and telemetry structures across services.

Responsibilities may include:
- request schema definitions
- response schema definitions
- event payload definitions
- metric payload definitions
- workload definitions
- validation structures
- serialization formats

The purpose of the shared schema layer is to keep data structures consistent across both benchmark paths and supporting infrastructure.
